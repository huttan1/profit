Litium Studio Product Catalog

Select “Control Panel” in the Litium Studio menu found in the top left corner. Then select “Product Catalog” in the left menu to display configuration options for Litium Studio Product Catalog.

General settings

This area allows you to configure general settings related to Litium Studio Product Catalog.

 

  • Use SKU - Specify if it is required to define the stock keeping unit when creating an article. If this checkbox is not selected you will not be able to define stocking keeping units when creating an article.
  • Add products with zero stock balances to price agents list - Specify if the products which are not in stock should be sent to the price agents list. If this checkbox is selected all products irrespective if there are in stock or not will be sent to the price agent lists. If this checkbox is not selected Litium Studio will only add the products which are currently available in your warehouses .
  • Reload cache - Possibility to reload cache. Should only be used by system administrator because it will affect the performance

 

Click “Save” to save modifications

Display templates

A display template is the template that will be used when viewing an product or a product group on the website. That is pretty much the same as a regular page template.

Add display template
  1. Click “New display template” and you will be taken to the “Add new display template” guide.

Enter following details.

  • Name - Name given for the template.
  • Type - For which entities this template can be used. Article or product group.
  • File name - The path to template that should be used.
  • Path to thumbnail - The path to the thumbnail that are displayed next to the template.
Edit display template
  1. Select the template and click “edit”.

Enter following details.

  • Name - Name given for the template.
  • Type - For which entities this template can be used. Article or product group.
  • File name - The path to template that should be used.
  • Path to thumbnail - The path to the thumbnail that are displayed next to the template.

Click “Save” to update details.

Template fields

Articles and product groups are created using field templates. This area contains the functionality to define these fields. One field can be used by several templates.

Add template field
  1. Click “New template field” and you will be taken to the “Add new field” guide.

Enter following details.

  • Field name - Name given for the template field.
  • Field translations - All languages added from Product Catalog language settings will be given here. Provide the language translations accordingly.
  • Field in import file: Name of the node in the XML file/or integration which corresponds to this particular field name.
  • Type - Refers to the data Type of the field. Select the required field type from Pre defined types listed in the combo. Available pre defined types are Text short, text long, image, date, decimal, file and Yes/No.
  • Mandatory - Select this option to make this field mandatory. If you use this field in any template (article/product catalog /product group) you MUST specify a value for this field accordingly.
  • Internal field - Allows specifying if the field is to be used for administration purposes or should be available for all. If selected this article field will only be available for administration not for public.
  • Searchable - Select this option to enable search using this field. E.g. If you make Description a searchable field and use it in the article template you will be able to search articles using the article description.
  • Write protected - Select this option to make this filed disabled.
  • Multi language support for the value of the field - Select this option to enable multi language support for the field. Once enabled you will be able to provide translations for different languages. If unselected it will be considered as a common field.
  • Alternatives/default field - select this option to provide default values for this field so that at the time of use you do not need to fill the information manually. Once this option is selected an extra panel will be enabled as shown in the figure. Define alternative values together with the order in which they should be displayed.

 

 

Click “Save” to save the template field

Edit template field
  1. Select template field from the list.
  2. Click “Edit” and update information accordingly.

Click “Save” to update modifications.

Delete template field
  1. Select a template field from the list.
  2. Click “Delete”.
  3. Confirm that you actually want to remove by clicking “Ok” to confirmation message that follows.
  4. This will remove the selected field as well as all values permanently.
Export template field
  1. Select a template field from the list.
  2. Click “Export” and save the file.
Import template field
  1. Select a template field from the list.
  2. Click “Import”.
  3. Click “Browse” and attach the relevant file.
  4. Specify if existing values has to be updated by clicking the “Update existing values” checkbox.
  5. Click “Import” to import the selected file.

Templates

Templates tab allows you to set up the required templates for entities used in product catalog.

Add Template
  1. Click new template and you will be taken to the “Add new template” guide.

Enter following details.

  • Name - Name of the template.
  • Type - Type of the template. All available template types (Article or product group) are given in the drop down.
  • Description of the template - Description given for the template.
  • Template to display with - Select the required template from the template drop down. All display templates are given in the drop down.
  • Add relations - Select a relation from the drop down and click Add. Relations created in Product Catalog Relations Settings will be given in the drop down.
  • Add template field - Select the required template field from the drop down and click Add. The selected template field will be displayed with its field characteristics. Template fields of field type Text Short will have an extra column as Display Name. By default the initially added field will be selected as the display name. You can simply specify which one of these fields should be used as the display name by checking the option button given in line with the selected field.
    Variant group fields - Select this option to make this field common for all articles inside a variant group E.g. Product name.
    Each template field added will be given a default sorting order. The sorting order basically refers to the order in which the fields are displayed in the Web Page. Modify the sorting order accordingly.
    You can also specify how information should be grouped in the Web Page simply by specifying a value in the group field.

Click “Save” to save details.

Edit template
  1. Select a template from the list.
  2. Click “Edit”.
  3. Do the necessary modifications.

Click “Save” to save detail.

Export template
  1. Select a template from the list.
  2. Click “Export” and save the file.
Import template
  1. Select a template from the list.
  2. Click “Import”. You will be directed to the following screen.
  3. Click “Browse” and attach the xml file to be imported.
  4. If importing already existing data specify if the existing data has to be updated by selecting the Update existing values checkbox.

Click “Import”.

Relations

Relations between product and product groups facilitate up sale and cross sale between products. E.g. "Product A is an accessory of Product B", or Product C is a part of Product D series.

Add new relation type
  1. Click New Relation type. You will be directed to “Add new relation type” screen.

Enter following details.

  • Relation type - Name of the relation.
  • Language translations - Provide translations for different languages.
  • Relationship type - Select a relationship type.
Edit relations
  1. Select a relation from the list.
  2. Click “Edit”. You will be directed to the change relationship type page.
  3. Do the necessary modifications.

Click “Save” to update details.

Delete telations
  1. Select a Relation from the list.
  2. Click “Delete”.
  3. Confirm that you actually want to remove the Relation by clicking Ok to confirmation message that follows. The selected Relation will be removed permanently.

Stock Keeping Units (SKU)

Stock keeping unit also known as SKU is the measurement unit of articles. Litium Studio uses this concept to measure and track quantities of your products/Articles.

Add stock keeping unit
  1. Click “Add” .

Enter following details.

  • Code for unit - Specify code for the unit. Can be more of a technical term.
  • Decimals to use for unit - When defining SKUs' it is important that you define the number of decimals allowed for that particular unit. Box of pencils can be a given a Decimals to use for unit as 0 as it is not practical to have box of pencils in decimals. Kilograms can be given a Decimal to use as 2 as it can be used as 5.50 kg, 7.50 kg etc
  • Description - Description of the stock keeping unit. All languages added from the language module will be given in this screen. Therefore you can give descriptions for different languages here.

Click “Save”.

View stock keeping unit
  1. Select the required Stock Keeping unit from the list.
  2. Click “View”.
Edit stock keeping units
  1. Select a Stock Keeping unit from the list.
  2. Click “Edit” and do the necessary modifications.

 

Click “Save” to save details.

Remove stock keeping unit
  1. Select a Stock Keeping Unit from the list.
  2. Click Remove.
  3. Confirm that you actually want to remove the Stock Keep Unit by clicking Ok to confirmation message that follows. You can't delete stock keeping units currently used in the application.

Import articles

This feature allows you to import articles from external sources.

Enter following details

  • Template - Select an article template from the drop down.
  • XML node containing the article - Refer following example, "Article".
  • Element containing the article number - Refer ArticleNumber Specify the XML-element that contains article number here. It is very important that you provide the correct number here as Litium Studio uses it to decide if it should create the article or update existing article details.
  • Element containing the url - Refer Description Specify the XML-element that contains the information that should be used for URl.
  • Language - Select language/languages which you require to import your articles by clicking the checkbox given next to the selected language
  • Select the file to import from: Click Browse and Attach the relevant file.

<Article>

<ArticleNumber>1485010</ ArticleNumber >

<Description>Frida chair black </ Description >

<Price>4,195,-</Price>

</Article>

<Article>

<ArticleNumber>1485011</ ArticleNumber >

<Description>Frida chair white </ Description >

<Price>4,195,-</Price>

</Article>

</Articles>

 

Click “Import” to import articles.
