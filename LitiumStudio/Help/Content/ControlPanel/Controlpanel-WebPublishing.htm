Litium Studio Web Publishing

Select “Control Panel” in the Litium Studio menu found in the top left corner. Then select “Web Publishing” in the left menu to display configuration options for Litium Studio Web Publishing.

Website

This area contains all configuration regarding websites inside Litium Studio Web Publishing. Information regarding Google Analytics, Enecto ProspectFinder and domains are all part of this configuration.

Create new website
  1. Click “New website” found in the toolbar.

Enter following information:

  • Standard Website - Specify if the newly created website is to be used as the default website by clicking this checkbox. There can be only one standard website.
  • Select language - Select site language from here. All available languages will be given in the drop down in the alphabetical order along with its country and flag. Website language is used for translations and language specific configurations.
  • Name - Name of the website.
  • Currency - Select which currency that should be used on this website.
  • Framework - Specify framework (Asp.Net masterpage) used as default framework for the website.
  • Domain name: Specify domain name of the site. It needs to be unique all over the solution. E.g.www.litium.se.
  • Google Analytics script - Litium Studio uses Google Analytics to analyze website traffic. It provides details about where your visitors come from and how they interact with your site. Add user information from Google Analytics here.
  • Activate page responsibility: Specify if “Page responsibility” has to be defined when creating web pages in this site by clicking the Active Page Responsibility checkbox.
  • Page permission step active - Specify if “Page permission” step should be visible when creating a new page.
  • Prospect finder: Obtain the Prospect Finder username and password from your administrator.

 

Click “Save”.

Click “Cancel” to cancel modifications.

Edit website
  1. Select a website from the list.
  2. Click “Edit”.

Edit details accordingly.

 

Click “Save”.

Click “Cancel” to cancel modifications.

Delete website
  1. Select the site which you want to remove.
  2. Click “Delete”.
  3. Click “Ok” to the confirmation message that following message.

Website Text

Texts which are displayed on the website (lable text,button text,etc) are considered as messages. Once you create a site you should either define new strings or import the relevent strings from another site. Messages created here will not be displayed in the interface directly. It has to be referenced to the relevent user control in the application code. Hence creation of new messages should be ideally done by an adminstrator.

Create new messages
  1. Select the website which you want to define website message.
  2. Click “Text”.
  3. Click New Text.

Enter follwing information.

  • Name - Name of the message.
  • Messages - The value of message.

 

Click “Save”

Edit texts
  1. Select the website which you want to edit messages.
  2. Click “Text”.
  3. Select the message which you want edit.
  4. Click Edit.
  5. Modify the message accordingly.

 

Click Save.

Delete texts
  1. Select the website which you want to delete messages.
  2. Click “Text”.
  3. Select the message which you want to delete.
  4. Click “Delete”.
  5. Click “Ok” to the confirmation message that follows. The select message will be removed from the site permanently.
Export texts
  1. Select the site which you want to export messages.
  2. Click “Texts”.
  3. Click “Export” and all messages will now be exported as an XML-file.
Import Texts
  1. Select the website which you want import website messages to.
  2. Click “Texts”.
  3. Click “Import”.
  4. Now click “Browse” and upload the file to be imported.
  5. Click “Import”.

Page types and templates

Presentation of information on your web site depends on the page types and page templates used in creating its web pages.

 

Page types defines the page content. Each page type is programmed to handle a unique set of functions related to a particular page type. Each page type contains different settings as well. It determines if a page can be visible in menu, if it can be printed etc.

 

Page pemplates provide the page layout, design and formatting information contributing to the final look and feel of the template. For example template of a page type defines the design and logic of your web page is arranged. Templates ensure feasible website maintenance and consistency across the site.

Create new page type
  1. Click “New page type”.

Enter following details

  • Name - Name of the page type. E.g. Newsletter, LoginPage.
  • Name language defined - All languages enabled in the application will be listed here. Provide language translations for the template name in the relevant fields.
  • Page properties: Page properties will be listed in the following manner. Enable the required properties by selecting the checkbox given in front of the property name. Available page properties are.

    Can be moved to archive - Are users allowed to archive this page manually?

    Auto Archive in days: Should the page be archived automatically? If YES specify archiving frequency in the text field.

    Can be moved to trashcan - Are users allowed to move the page to the trashcan?

    Can be master page - Are users allowed to add this page as a master page when creating a shared page?

    Can be in menu - Should this page be displayed in the menu.

    Can be in site map - Should this page be displayed in the site map.

    Possible to link to the page - Should the user be able to create internal links to this page.

    Is printable - Should the user be given permission to print this page?

    Is Searchable - Should the user be able to search the content in this page.

    Create versions, number of versions to keep: Should versions be created for this page type. If YES specify maximum number of versions in the text field.

    Save visit statistics - Should Litium Studio keep track of visitor statistics.

    Content is editable - Should the user be able to edit page content.
  • Possible child page types: All available pages will be listed here. From these select the page/pages which should be permitted to be added as child page under current page type.
  • Possible parent pages types - All available pages will be listed here. From these select the pages which should be can be parent page to pages of this page type.
  • Page type properties - Available properties are:
    Edit page - Allows you to replace the built in functionality when editing/creating a CMS page. Specify the path as ~/Site/CMS/Templates/EditPages/EditPage.aspx.

    Path to user defined page content - To extend the Text tab by your own code, it is possible to use external content file. Specify the path to the file here. An ascx-file.

    Path to user defined page settings - Similarly you can also externd settings from an external settings file. Specify the path to the page setting file here. An ascx-file.

    This page is deletable - Select this option to enable users to delete this page

 

Click “Save” to save the page type.

Click “Cancel” to cancel creation of the page type.

Add/Edit content fields

Content fields are those who are found under the tabs “Text”, “Images” and “Files”. The value of these fields are searchable and are used on the public website.

 

  1. Select the page type which you want to add/edit content fields.
  2. Click “Content fields”. You will be taken to the edit content properties page.

Enter following details.

  • Name - Name of the field.
  • Type - Type of the field. E.g. Text, Image.
  • Click “Add field” button. The added field will be displayed in the grid below.
  • Specify if fields are mandatory, Arrays and are to be shown in the Edit mode.
Add/Edit settings fields

Settings fields are those who are found under the tabs “Settings”. The value of these fields are not searchable and are primary used for configuration.

 

  1. Select the page type which you want to add/edit content fields.
  2. Click “Settings fields”. You will be taken to the edit settings properties page.

Enter following details.

  • Name - Name of the field.
  • Type - Type of the field. E.g. Text, Image.
  • Click “Add field” button. The added field will be displayed in the grid below.
  • Specify if fields are mandatory, Arrays and are to be shown in the Edit mode.
Edit page type
  1. Select the page type which you want to modify.
  2. Click “Edit”. You will be taken to the Edit page Type page.
  3. Modify details.

 

Click “Save” to save modifications.

Click “Cancel” to cancel modifications.

Import page types
  1. Select the page type which you want to Import.
  2. Click Import. You will be taken to the “Import page type” page.
  3. Click “Browse” and attach the file to be imported.
  4. Click Import to upload the attached field.
Export page types

By exporting a page type it makes its possible to use page types in other projects.

  1. Select the page type which you want to export.
  2. Click “Export”.
  3. Exported file will be opened/saved.
Add a template to existing page type
  1. Select the page type which you want to add new templates.
  2. Click “Template” found in toolbar
  3. Click “New Template” in toolbar. You will be taken to “Create template” page.

Enter following details.

  • Name - Name of the template.
  • Filename - File name of the template. Path based from Templates folder. E.g. Article1.aspx.
  • Path to thumbnail - Path to the preview image of the selected template. E.g. Site/CMS/Templates/Article/Images/Article1.gif

Click “Save” to save details.

Edit template
  1. Select the page type which you want to edit templates.
  2. Click “Templates” and all templates will be listed.
  3. Select the template which you wish to modify and click “Edit”.

 

Click “Save” to save details.

Delete template
  1. Select the page type from which you want to delete the page template.
  2. Click “Templates” and all page templates will be listed.
  3. Select the template which you want delete.

 

Click “Delete”. The selected template will be removed permanently.

Page responsibility

Page responsibility tab allows you to view /edit/decline page responsibility assigned to persons for web pages (Web pages of all available sites) created in Litium Studio Web Publishing.

View page responsibility
  1. Select “Page responsibility” found in the left menu.
  2. All persons who is assigned page responsibility will be listed with the total number of pages and contact email address.
View pages without page responsibility
  1. Select “Page responsibility” found in the left menu.
  2. Click “Show pages without page responsibility”, found in the toolbar, and all pages which have not been assigned a responsible person will be listed here.
Set page responsibility
  1. Select “Page responsibility” found in the left menu.
  2. Click “Show pages without page responsibility” and all pages which have not been assigned a responsible person will be listed here.
  3. Select a page from the list.
  4. Click “Set responsibility”.
  5. Select the person which you want to assign for the selected page.

 

Click “Save”.

 

The particular page will be assigned to the selected person and will be displayed in My Task: My Page Responsibility panel of the selected person.

Decline responsibility
  1. Select “Page responsibility” found in the left menu.
  2. Select a person from the list and click “Change” and all pages assigned to the selected person will be displayed.
  3. Select the page which you want decline responsibility.
  4. Click “Decline responsibility”.
  5. Click “Ok” to the confirmation message that follows.
Maintenance

All general settings related to website maintenance are listed in this area.

 

Delete versions

To delete all versions of a given website, following these steps:

  1. Select the website which you want to delete versions from “Delete versions” panel. All websites created in Litium Studio will be given here.
  2. Click “Delete versions” button given in line with the selected site.

 

The above method will clear version history of all web pages of the selected site.

Delete visitor statistics

Litium Studio also monitors visitors to your site and provides you comprehensive visitor statistics In order to clear this history:

 

  1. Specify the date to which you want to clear visitor statistics
  2. Click “Delete”. All visitor statistics older than the selected date will be removed immediately. The above approach will delete visitor statistics of all the available websites.
Empty trashcan

Click “Delete” given in the Empty Trashcan panel to remove all pages existing in the trashcans. Above step will clear trashcans of all existing sites.

Empty archives

Click “Delete” given in the “Empty archives” panel to delete all pages in the website archive. Above step will clear Archive folders of all existing sites.
