Working with content

Add a page

When adding a page, the first choice will be to determine what type of page you will use. There are several types of pages to use:

  • Regular page - Common content page for image and text. Technical known as page type “Article”.
  • Product catalog page - Specific page types that are used to display product information.
  • Function page - All other page types, such as contact form, shopping cart etc. The number of page types depends on your license.
  • Shared page - Are used to share content between websites. The content will be the same, but templates, framework, URL and search words are individually.
  • Translation - Create a translation work flow based on a specific page. Each time a page is updated, the translator will be notified.

 

 

  1. Select the website, from which this new page will be created.
  2. Highlight the page (1) that will be parent page to the new page. When adding as a sub page, only relevant page types are available to the selected node (page). This is a setting per page type. E.g. you will only be able to add a “News” page to a “News list” page.
  3. Click “New” (2), found in the toolbar and a drop down menu will be shown. All possible types will be shown.
  4. In this scenario, select “Regular page”.

 

Follow steps found in this guide to create a new page

Add a page - Step 1

Enter following details

  • Page name - This is the name which will be used by Litium Studio. Name the web page to suit its content so that your customers can easily find the required information they need. The Page Name should be descriptive and short.
  • URL name - Define the name that will be displayed in browser to identify the particular page. The whole URL will be built based on top of the parent page URL. By default, the URL name will be formated in lower case and - instead of space. It is possible to change this. In the screen shoot, the URL of the page “Pagetypes” will be www.litium.com/information/pagetypes.
  • Page templates - Page templates define the design in which the template content (text, images and files) will be presented in the web page. It is possible to change later. Select the template which matches your requirement. Page templates are added during installation.

 

Click “To the next step” to proceed to Step 2 of the wizard

Click “Cancel” to exit the wizard .

 

Add a page - Step 2

Step 2 of the wizard allows you to set page responsibility. This step will only be shown if the web site has “Page responsibility” setting defined to true. This can found under settings for current web site.

All users with administrative rights will be given in the “Page responsibility” table. Select the required person from this list.

 

Click “To the next step” to proceed to Step 3 of the wizard.

Click “Back” to previous step to return to Step 1.

 

Add a page - Step 3

This step allows you to set page permission for individual user groups to read/write/publish this. This step will only be shown if the web site has “Permissions” setting defined to true.

 

By default any new page will inherit user group permission from its parent page. Therefore when creating web pages with same permissions you can simply skip this step. But if required you can also add/edit permissions from here.

 

All available groups will be given in the left grid.

  1. Select the user group which needs to be given access to the page and click on the right arrow.
  2. Specify permission by selecting the Read, Write and Publish checkboxes given in line with the selected user group
  3. To remove a selected group, click on the left arrow, that particular group will be moved back to the “Available groups” panel.

 

By default, user groups with full permission to all functions in the module will be given in the right panel. These user groups will have all Read, Write and publish checkboxes selected.

Unlike other groups you will not be able to remove full permission groups manually

 

Click “Back” to previous step to return to the previous step.

Click “Save” to save the web page. The newly created web page will be added as an unpublished and blank page.

Click “Save and edit” content to save the web page and to be directed to the working copy of the page. Read more about how to work with a page on the next page, See Edit a page.

Search words tab

This tab allows you to specify search related functions of this page.

  • Title - Title refers to the text displayed in the web browser.
  • Description - The text given here will be displayed as search result text when searched using a search engine. Customers' decision on visiting the site depends on the page description given here. Hence it is necessary that you give an attractive description (of Maximum 100 characters spaces included) perfectly describing the content in the web page.
  • Key words - Keywords are search words connected to the web page. That is once a customer gives certain search criteria as search text the search engine will look search for web pages which has keywords similar to the words given as the search criterion by the customer. Therefore you should identify the best key words to represent the web page content. You can give multiple search words separated by commas starting with the most import words
  • Change frequency - Refers to how frequently the web page is likely to change. This field provides general information to search engines and may not correlate exactly to how often they crawl the page.
  • Priority - The priority of the URL of this web page relative to other URLs on your site. Valid values range from 0.0 to 1.0. This value has no effect on your pages compared to pages on other sites, and only lets the search engines know which of your pages you deem most important so they can order the crawl of your pages in the way you would most like.

Publishing options

Once the page has been created/edited it has to be published before it can be accessed by a visitor.

 

Following publishing options occurs when editing a page.

 

 

 

 

  • Publish immediately - Publish immediately will publish the selected page immediately. Only authorized users will be able to publish web pages.
  • Ready to publish - If current editor do not have permissions to publish the page it has to be sent to the authorized user. This option is used to assign the page for approval. Once a page is made ready to publish any user with full permission to the web publishing module can approve and publish the page.
  • Time publish: This option allows the you to specify a time and a date to publish the page. Once this option is selected following calendar will be given for you to specify date and time to publish the selected page. You may cancel a time publish later and reschedule it to a newer time.

Superscript text

Characters which are half the height of normal characters and placed above the other characters are known as superscript text. (E.g. 12th December)

To create a superscript text, select the text you want format as superscript and click “Superscript”(t).

 

Subscript text

Characters which are half the height of normal text and are placed below the bottom line of text are known as subscript text. (E.g. X3)

 

To create a subscript text, select the text you want format as subscript and click “Subscript”(u).

Tables

Tables are used as a means of attractive data presentation. Tables are mainly used to present information which has a logical relationship within the data.

 

Create new table

  1. Move your mouse pointer to the place you want to have your table and click “Insert/Edit table” (l) .
  2. “Table properties” popup will be shown.

 

 

 

 

  • Rows - Refers to the number of horizontal lines in the table. Enter the required number of rows in the text box.
  • Columns - Refers to vertical sections in the table. Enter the required number of columns in the text box.
  • Height - Cell height is the vertical length of a table cell. Define the required cell height in the textbox.
  • Width - Cell width refers to the horizontal length of a cell. The default value of the cell width will be given as 200 pixels. You can edit this value accordingly.
  • Cell spacing - Cell spacing is the distance between the cells in the table. By default this value will be given as one. You can edit this value accordingly
  • Cell padding - Cell padding is the distance from the edge of the cell to the content. By default this value will be given as one. You can edit this value according to your requirement.
  • Border size - Border size is the thickness of the table borders.
  • Alignment - This field allows you to specify the alignment (Left, Right or center) of the table in the web page content. Select the required option from the Alignment drop down.
  • Caption - Specify the table heading here.
  • Summary - Specify a brief summary of the table content here. Description specified here will not be displayed in the editor but it will be displayed in the text editor source.

 

Finally click “Ok” to create the table according to the specified dimensions

Anchor link

In order to link content in the same page you can make use of an anchor. The anchor as the name applies marks the position of selected text in the content.

 

Linking of text in the same page is done in two steps.

 

Step 1 - Construct anchor

  1. Select the text which needs to be tagged as an anchor.
  2. Click “Insert/Edit anchor” (y).
  3. Specify an Anchor name from the “Anchor properties” popup will be shown.
  4. Click Ok to add the anchor.

 

Now that you have created the anchor you have to link it to the relevant text.

 

Step 2: Tag anchor to word

  1. Select the word to be linked to the anchor
  2. Click “Insert/Edit link” (z).
  3. Go to “Link info” tab.
  4. Select “Link to Anchor in the text” as the link type.
  5. Select the relevant anchor name from the “By Anchor name” drop down. All anchors created in this page will be given in the drop down

 

Delete anchor

Select the link that should be removed and right click. Click “Remove anchor” from the menu.

Paste content

Text editor provides you various paste options to facilitate transferring of text from the same text editor as well as external document

Paste - Paste option most suitable for pasting copied/cut content of the same text editor. This option will copy the content with formatting.

Paste as plain text - Paste as plain text option removes all formatting when copying content.

 

Paste from word - Paste option most suitable for pasting content taken from MS Word.
To Paste content from Word:

  1. Select content from a word document
  2. Click Cut or copy.
  3. Move the cursor to the location where you want to paste the content and click “Paste from Word” (e). Unlike the above options here you will not be able to directly paste to the editor. Hence you will be given a popup where you will specify following options:

    Ignore font face definitions - Select this to remove the font setting. When this option is selected the text will be pasted in the font type used in the text editor.

Shared pages

A shared page is a specific type of page that shares content from another page. The idea is to use the same content in another position, but act as a individual page. The new page has its own search words, template, framework and URL. A common scenario is to use this functionality to share company information between two english websites. It is only needed to update the information in one place, but they will act as two standalone web pages.

 

Create a shared page

 

  1. Highlight the page which should be parent page to the new page.
  2. Select “New”, found in the toolbar and click on “Shared page”.

 

You will be directed to the “Create new page” wizard.

 

Create a shared page - Step 1

Enter following details

  • Master page - Master page refers to the page which you are going to share/get content from. All the sites currently available in the site will be listed with their sub pages. Select the page you want to share from the tree. Following pages in the tree will be disabled and because of that it is not possible to share their content:
    Pages which page types has disabled the possiblity to act as a “Master page”.
    Unpublished pages
    Pages created by sharing pages
  • Include sub page: Specify if the sub pages attached to the selected page should also be shared. If sub pages need to be shared, select all sub pages of the chosen page are copied as shared pages checkbox

 

 

Click “To the next step” to proceed to Step 2

Click “Cancel” to exceed the wizard

 

 

Create a shared page - Step 2

Enter following details

  • Page name - This is not possible to edit, it is a part of the shared page.
  • URL name - Define the name that will be displayed in browser to identify the particular page. The whole URL will be built based on top of the parent page URL. By default, the URL name will be formated in lower case and - instead of space. It is possible to change this. In the screen shoot, the URL of the page “Pagetypes” will be www.litium.com/information/pagetypes.
  • Page templates - Page templates define the design in which the template content (text, images and files) will be presented in the web page. It is possible to change later. Select the template which matches your requirement. Page templates are added during installation.

 

Click “To the next step” to proceed to Step 3 of the wizard

Click “Cancel” to exit the wizard .

 

 

Create a shared page - Step 3

Step 3 of the wizard allows you to set page responsibility. This step will only be shown if the web site has “Page responsibility” setting defined to true. This can found under settings for current web site.

All users with administrative rights will be given in the “Page responsibility” table. Select the required person from this list.

 

 

Click “To the next step” to proceed to Step 4 of the wizard.

Click “Back” to previous step to return to Step 2.

 

 

Create a shared page - Step 4

This step allows you to set page permission for individual user groups to read/write/publish this. This step will only be shown if the web site has “Permissions” setting defined to true.

 

By default any new page will inherit user group permission from its parent page. Therefore when creating web pages with same permissions you can simply skip this step. But if required you can also add/edit permissions from here.

 

 

All available groups will be given in the left grid.

  1. Select the user group which needs to be given access to the page and click on the right arrow.
  2. Specify permission by selecting the Read, Write and Publish checkboxes given in line with the selected user group
  3. To remove a selected group, click on the left arrow, that particular group will be moved back to the “Available groups” panel.

 

 

By default, user groups with full permission to all functions in the module will be given in the right panel. These user groups will have all Read, Write and publish checkboxes selected.

Unlike other groups you will not be able to remove full permission groups manually

 

 

Click “Back” to previous step to return to the previous step.

Click “Save” to save the web page. The newly created web page will be added as an unpublished and blank page.

Click “Save and edit” content to save the web page and to be directed to the working copy of the page. Read more about how to work with a page on the next page, See Edit a page.

 

Edit a shared page
It exists two different ways to edit a page. One is to right click on the page that should be edited, and select “Edit” from the menu. The other is to select the page that should be edited and click “Edit” found in the toolbar.

 

When reaching the page edit guide, three different tabs will be displayed.

 

 

Shared page tab

Shared page tab provides you details of the master page to which the selected page is linked to.

It will display the name of the master page, location and the day on which the shared page was created

In order to view the page click blue arrow icon in line with the page details

 

Search words tab

This tab allows you to specify search related functions of this page.

  • Title - Title refers to the text displayed in the web browser.
  • Description - The text given here will be displayed as search result text when searched using a search engine. Customers' decision on visiting the site depends on the page description given here. Hence it is necessary that you give an attractive description (of Maximum 100 characters spaces included) perfectly describing the content in the web page.
  • Key words - Keywords are search words connected to the web page. That is once a customer gives certain search criteria as search text the search engine will look search for web pages which has keywords similar to the words given as the search criterion by the customer. Therefore you should identify the best key words to represent the web page content. You can give multiple search words separated by commas starting with the most import words
  • Change frequency - Refers to how frequently the web page is likely to change. This field provides general information to search engines and may not correlate exactly to how often they crawl the page.
  • Priority - The priority of the URL of this web page relative to other URLs on your site. Valid values range from 0.0 to 1.0. This value has no effect on your pages compared to pages on other sites, and only lets the search engines know which of your pages you deem most important so they can order the crawl of your pages in the way you would most like.

 

Templates tab

Template tab displays all templates related to current web page. By default the template selected during web page creation will be selected but it is possible to change the template in real-time.

 

Click “Preview” to preview the content in the page.

Click “Save” to save the modifications but it will not be published.

Click “Publish” to publish information. See next chapter for more information.

Click “Cancel” to cancel the changes done to the page.

Click “Cancel and delete working copy” to cancel changes and delete the working copy.

Product catalog page

A product catalog page is a dedicated page type to handle product information.

The purpose with a product catalog page is to get product information from Litium Studio Product Catalog. This kind of page type does not have a template connected to itself, instead it is the display template of the current product group that will be shown. But, of course, the display template can display any kind of page information defined in the product catalog page.

 

Example

The page “Products” has a connection to “Web assortment”, inside Litium Studio Product Catalog. The page are using the display template, defined in the field template of “Web assortment”.

 

 

 

 

Add a product catalog page
  1. Highlight the page which should be parent page to the new page.
  2. Select “New”, found in the toolbar, select “Product catalog page” and click on prefered page type. If the selection is gray, make sure that it exists page types with its type defined to “Product catalog” and that it exists display templates associated with field templates inside Litium Studio Product Catalog.
