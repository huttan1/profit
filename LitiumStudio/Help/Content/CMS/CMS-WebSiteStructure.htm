Web sites

A web site can be a public site, intranet or a language site but the functionality are all the same in Litium Studio. A web site are displayed as a blue globe in the tree structure. A page structure, a trashcan, a status area, an archive and visit statistic are shown for each web site.

 

 

Pages

A web page is the foundation for Litium Studio Web Publishing and holds the content and design. In Litium Studio all content, structure and design are separated for easier handling and more flexibility for the editor to work with information.

Every website consists of a start page and sub pages. In the example above, the page called “Home” will be start page and its URL are connected to the root of the UK domain. For example www.litium.co.uk/.

  2. Move the selected page to trashcan
  3. Cut the selected page along with its subpages
  4. Copy the selected page
  5. Copy the selected page with its subpages

Lock and unlock a page

When you are working on a web page you may want to ensure that any other user will not be working on the same copy as you are. This can be done using the lock feature available in the web publishing module.

Even though a page is locked if you have appropriate responsibility you can unlock and continue working it.

 

In order to lock a page

  1. Select a web page from the tree. The highlighted page will be displayed in the right panel
  2. Click “Lock”

 

The selected page will be locked and a new menu option “Unlock” will be given.

If another user access this page using another login Edit and Hide in the menu will be disabled and Unlock option will be displayed in the menu.

 

Hide page

When a page is hidden it won’t be visible in menu but it will still be accessable through browsing to its URL. Some page types might have a default setting that hides all pages of this page type. When this setting occurs, it will not be possible to edit that page.

 

To hide a page

  1. Select a web page from the tree. The highlighted page will be displayed in the right panel
  2. Click “Hide in the menu”

The links to access the selected page will be removed from all the pages and the tree node displaying that particular page will be displayed with a dotted line

 

Settings

Settings menu can be used to set up general settings related to a web page; including page permission, page responsibility, and version control and advance settings. You will reach the settings for current page by right click on current page and select “Settings”

 

Permissions

Permissions option available in “Settings” allows you to edit group permissions set during creation of the web page.

 

All available groups will be given in the left grid.

  1. Select the user group which needs to be given access to the page and click on the right arrow.
  2. Specify permission by selecting the Read, Write and Publish checkboxes given in line with the selected user group
  3. To remove a selected group, click on the left arrow, that particular group will be moved back to the “Available groups” panel.

 

 

By default, user groups with full permission to all functions in the module will be given in the right panel. These user groups will have all Read, Write and publish checkboxes selected.

Unlike other groups you will not be able to remove full permission groups manually

 

Page responsibility

Responsibility option available in “Settings” allows you to edit page responsibility set during creation of the web page.

 

Version list

Version list option given in settings tracks the publishing history of a selected page.

Creation of page versions will be done ONLY if the page property "Create Versions" is enabled.

View a version
  1. Select a web page from the tree. The highlighted page will be displayed in the right panel
  2. Click “Settings” and select “Version list” from the drop down menu.
    You will be directed to the version list page. All available versions will be listed with a time stamp.
  3. Select the required version from the list
  4. Click “Display”
    The selected page will be displayed as how it is displayed when published

 

Click “Restore” to restore the page

Click “Back” to return to the version list page

 

Restore version

Every time you publish a page, Litium Studio will keep a version of the changes (Creates a version) and list them in the version list page. As a result at any time you can easily reverse back to a desired version.

  1. Select a web page from the tree. The highlighted page will be displayed in the right panel
  2. Click “Settings” and select “Version list” from the drop down menu.
    You will be directed to the version list page. All available versions will be listed with a time stamp.
  3. Select the required version from the list
  4. Click “Restore”.

 

Click “Ok” to the confirmation message that follows. The selected version will be restored and the remaining versions will be deleted

 

Advanced

Advanced settings contains information about the current page.

  • Address - Displays the URL to current page.
  • Page type - The name of current page type is displayed.
  • Template - Which page template are used.
  • Start page - It is possible to define any page as a start page. It can be usefull if you temporary want to display a campaign page as start page.
  • Deactivate in menu - Deactivating means that the page is displayed but not clickable in the menu.
  • Sitemap - Defines whether the page is displayed in the sitemap (Even Google Sitemap).
  • Unpublish date - If you want to define a specific date to unpublish this page.
  • Auto archive - Auto archive means that the page is transferred to the archive at a given time.

 

Working copy

Every time you are going to work with a page a working copy will automatically be created for you to work with, this instead of working with the content on the published page. This will make It possible for you to temporary save your work instead of working on the real page. View working copy allows you to preview how the content will be presented before actually publishing the page.

 

 

 

In order to view working
  1. Select a page with a working copy
  2. Click “Working copy” found in the toolbar.
  3. The selected page will be displayed in the following manner
    Click “Edit” to edit the working Copy
    Click “Delete working copy” to delete the working copy
    Click “Publish” to publish the working copy.

Archive

Archive a page
  1. Select the page which you want to archive
  2. Drag and drop it to the “Archive” node given for its website.
  3. Click “Ok” to the confirmation message that follows. The selected page will be archived.
    You can archive a web page only if “Can be moved to archive” page property is set to true in the selected page type.
    If you have enabled the “Auto Archive” option, Litium Studio will automatically archive the page according to the set time interval.
View archived pages
  1. Click “Archive” node
    You will be able to view all pages archived pages in the selected
Restore archived page
  1. Click “Archive” node .
    All archived pages will be displayed.
  2. Select the page which you want to restore
  3. Click “Restore” .The selected page will be restored to its original position as an unpublished page

Trashcan

Every page that is deleted will first be stored in the trashcan until you clear the trashcan.

Move a page to trashcan
  1. Select the page which you want to delete
  2. Drag and drop it to the trashcan node given for its website or click “Move to trashcan” option given in the menu or click “Move to trashcan” found when you right click on the page..
  3. Click “Ok” to the confirmation message that follows. The selected page will be deleted from the site
    You can move a web page to the trashcan only if “Can be moved to trashcan” page property is set true in the selected page.

 

View deleted pages

Litium Studio stores all deleted pages for future reference.

  1. Click “Trashcan” node .
    You will be able to view all pages deleted from the selected site

 

Restore deleted page
  1. Click “Trashcan”.
    All deleted pages currently existing in the website will be displayed.
  2. Select the page which you want to restore
  3. Click “Restore” .The selected page will be restored to its original position as an unpublished page

 

Status

Status option available in the web site tree provides you comprehensive details about the current status of the website including the total number of pages, total number of published/unpublished pages, achieved page, etc

In order to view summary
  1. Chose the website which you require to view the summary
  2. Click the “Status” node given in the tree

 

Visit statistics

Statistics tab provides you a comprehensive analysis of visitor statistics related to a selected page

In order to view statistics
  1. Click “Visit statistic” node.
  2. Following information will be shown:

    Google analytics - If you hace associated a Google Analytics account to this website, a link to Google Analytics will be shown.

    Landing page - Gives the statistics as to how many times the selected page was the customers first choice (the first page viewed by a visitor on the page)

    Exit page : Gives the statistics as to how many times the selected page was the last page viewed by the customers
